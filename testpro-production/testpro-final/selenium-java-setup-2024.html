Prerequisites & What You'll Need

Before we start, make sure you have the following. Don't worry — everything listed here is free:

  • A Windows, Mac, or Linux machine (Windows 10/11 recommended for beginners)
  • Stable internet connection to download tools
  • Basic understanding of Java syntax (variables, methods, classes)
  • Around 2 GB of free disk space

If you're completely new to Java, spend one week on a free Java basics course on YouTube first. Then come back here — this guide will make much more sense.

Step 1 — Install Java JDK

Selenium requires Java to be installed on your machine. We'll use Java 17 LTS (Long Term Support), which is stable and well-supported.

1Go to adoptium.net and download the Temurin JDK 17 for your OS
2Run the installer. On Windows, check "Set JAVA_HOME variable" during installation
3Verify installation — open Command Prompt and run:
java -version
# Expected output:
# openjdk version "17.0.x" ...
⚠️ Common Mistake Many beginners download Java from oracle.com and face licensing issues. Use Adoptium (Eclipse Temurin) — it's 100% free and open-source.

Step 2 — Install Eclipse IDE

Eclipse is the most popular IDE for Selenium automation in India's IT industry. It's free and what most companies use.

1Go to eclipse.org/downloads and download "Eclipse IDE for Java Developers"
2Extract the zip and run eclipse.exe (Windows) or eclipse (Mac/Linux)
3Choose a workspace folder — e.g. C:\SeleniumWorkspace

Step 3 — Create a Maven Project

Maven manages your project dependencies automatically. Instead of downloading Selenium JARs manually, Maven downloads and updates them for you.

1In Eclipse: File → New → Maven Project
2Check "Create a simple project (skip archetype selection)"
3Fill in: Group ID: com.testpro, Artifact ID: selenium-project
4Click Finish. Eclipse creates the Maven project structure

Step 4 — Add Selenium Dependencies

Open the pom.xml file in your project. Replace its contents with the following — this adds Selenium 4, TestNG, and WebDriverManager (which handles ChromeDriver automatically):

<project xmlns="http://maven.apache.org/POM/4.0.0">
  <modelVersion>4.0.0</modelVersion>
  <groupId>com.testpro</groupId>
  <artifactId>selenium-project</artifactId>
  <version>1.0</version>

  <dependencies>
    <!-- Selenium WebDriver -->
    <dependency>
      <groupId>org.seleniumhq.selenium</groupId>
      <artifactId>selenium-java</artifactId>
      <version>4.15.0</version>
    </dependency>

    <!-- WebDriverManager - auto manages ChromeDriver -->
    <dependency>
      <groupId>io.github.bonigarcia</groupId>
      <artifactId>webdrivermanager</artifactId>
      <version>5.6.3</version>
    </dependency>

    <!-- TestNG -->
    <dependency>
      <groupId>org.testng</groupId>
      <artifactId>testng</artifactId>
      <version>7.8.0</version>
      <scope>test</scope>
    </dependency>
  </dependencies>
</project>

Save the file. Maven will automatically download all dependencies. This may take 2–5 minutes on first run.

Step 5 — Write Your First Test

Create a new Java class in src/test/java called FirstSeleniumTest.java:

import io.github.bonigarcia.wdm.WebDriverManager;
import org.openqa.selenium.By;
import org.openqa.selenium.WebDriver;
import org.openqa.selenium.WebElement;
import org.openqa.selenium.chrome.ChromeDriver;
import org.testng.Assert;
import org.testng.annotations.AfterMethod;
import org.testng.annotations.BeforeMethod;
import org.testng.annotations.Test;

public class FirstSeleniumTest {

    WebDriver driver;

    @BeforeMethod
    public void setUp() {
        // WebDriverManager handles ChromeDriver automatically
        WebDriverManager.chromedriver().setup();
        driver = new ChromeDriver();
        driver.manage().window().maximize();
    }

    @Test
    public void verifyGoogleTitle() {
        driver.get("https://www.google.com");

        // Verify the page title contains "Google"
        String title = driver.getTitle();
        Assert.assertTrue(title.contains("Google"),
            "Page title should contain 'Google' but was: " + title);

        System.out.println("Test PASSED! Title: " + title);
    }

    @AfterMethod
    public void tearDown() {
        if (driver != null) {
            driver.quit();
        }
    }
}

Step 6 — Run & Verify

Right-click the test class → Run As → TestNG Test. You should see:

  • Chrome browser opens automatically
  • Google.com loads
  • Browser closes after the test
  • TestNG console shows: Tests run: 1, Failures: 0
🎉 Congratulations! You've just run your first automated test. From here you can expand this to test login forms, search functionality, and entire user flows.

Common Errors & Fixes

1. "ChromeDriver version mismatch"

This used to be a common headache. Since we're using WebDriverManager, it auto-downloads the right version. If you still see this error, check that your internet connection is active when running tests.

2. "Cannot find symbol: WebDriverManager"

Maven hasn't downloaded dependencies yet. Right-click project → Maven → Update Project, or run mvn clean install in terminal.

3. Test runs but Chrome opens and immediately closes

Add Thread.sleep(3000); before driver.quit() temporarily to debug — this is almost always a timing issue in the test logic.

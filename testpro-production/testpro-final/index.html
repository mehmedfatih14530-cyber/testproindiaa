Interview Prep

Most-Asked QA Interview Questions

Curated from real interviews at Infosys, TCS, Wipro, Accenture, and product startups across India.

Q01What is the difference between Verification and Validation in software testing?
Verification checks whether we are building the product right — it's a static process involving reviews, walkthroughs, and inspections done without executing code. Validation checks whether we are building the right product — it's a dynamic process where the actual software is executed and tested against requirements. Tip: remember "Verification = process, Validation = product" to recall easily in interviews.
Q02Explain the Software Testing Life Cycle (STLC) with all phases.
STLC has 6 phases: (1) Requirement Analysis — understand what to test; (2) Test Planning — create test plan, estimate effort, assign roles; (3) Test Case Design — write test cases and prepare test data; (4) Test Environment Setup — prepare servers, browsers, devices; (5) Test Execution — run tests, log defects; (6) Test Closure — prepare metrics report, lessons learned. Entry and exit criteria are defined for each phase.
Q03What is the Page Object Model (POM) in Selenium? Why is it used?
POM is a design pattern in Selenium where each web page has a corresponding Java class. The class holds the web elements (locators) and methods (actions) for that page. Benefits: (1) Reduces code duplication — locators defined once; (2) Easy maintenance — if a locator changes, update only one file; (3) Improves readability — test scripts are cleaner and business-focused. In interviews, always mention you've used POM with PageFactory for cleaner initialization.
Q04How do you prioritize test cases when time is limited before a release?
I follow a risk-based approach: (1) Identify business-critical flows (login, payment, core feature); (2) Test areas that changed in this release (regression risk is highest here); (3) Prioritize tests that previously found high-severity bugs; (4) Cover smoke tests to ensure app stability before detailed testing. I also communicate with the product manager about which features are customer-facing for the upcoming release. This shows you think beyond just "running tests."
Q05What is the difference between Smoke Testing, Sanity Testing, and Regression Testing?
Smoke Testing: Shallow, wide — done after a new build to verify basic functionality works (will the app launch? can the user log in?). Sanity Testing: Narrow, deep — done after a bug fix to verify that specific fix works without breaking nearby features. It's a subset of regression. Regression Testing: Full re-run of existing test cases after any change to ensure nothing is broken. Smoke is done first; if it fails, the build is rejected without further testing.
